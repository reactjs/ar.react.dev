<Intro>

غالبًا تحتاج المكوّنات إلى تغيير ما يظهر على الشاشة نتيجة تفاعل. الكتابة في النموذج يجب أن تحدّث حقل الإدخال، والنقر على «next» في شريط صور يجب أن يغيّر الصورة المعروضة، والنقر على «buy» يجب أن يضيف منتجًا إلى سلة التسوق. تحتاج المكوّنات أن «تتذكّر» أشياء: قيمة الإدخال الحالية، الصورة الحالية، سلة التسوق. في React، تُسمّى هذه الذاكرة الخاصة بالمكوّن *حالة* (state).

</Intro>

<YouWillLearn>

* كيفية إضافة متغير حالة بخطاف [`useState`](/reference/react/useState)
* أي زوج من القيم يعيده خطاف `useState`
* كيفية إضافة أكثر من متغير حالة
* لماذا تُسمّى الحالة محلية

</YouWillLearn>

## عندما لا يكفي متغير عادي {/*when-a-regular-variable-isnt-enough*/}

إليك مكوّنًا يصيّر صورة منحوتة. النقر على زر «Next» يجب أن يعرض المنحوتة التالية بتغيير `index` إلى `1` ثم `2`، وهكذا. لكن هذا **لن يعمل** (يمكنك التجربة!):

<Sandpack>

```js {expectedErrors: {'react-compiler': [7]}}
import { sculptureList } from './data.js';

export default function Gallery() {
  let index = 0;

  function handleClick() {
    index = index + 1;
  }

  let sculpture = sculptureList[index];
  return (
    <>
      <button onClick={handleClick}>
        Next
      </button>
      <h2>
        <i>{sculpture.name} </i> 
        by {sculpture.artist}
      </h2>
      <h3>  
        ({index + 1} of {sculptureList.length})
      </h3>
      <img 
        src={sculpture.url} 
        alt={sculpture.alt}
      />
      <p>
        {sculpture.description}
      </p>
    </>
  );
}
```

معالج الحدث `handleClick` يحدّث متغيرًا محليًا `index`. لكن أمرين يمنعان ظهور ذلك التغيير:

1. **المتغيرات المحلية لا تبقى بين عمليات التصيير.** عندما يصيّر React هذا المكوّن مرة ثانية، يصيّره من الصفر—لا يعتبر تغييرات المتغيرات المحلية.
2. **تغييرات المتغيرات المحلية لا تُطلِق تصييرًا.** لا يدرك React أنه يجب إعادة تصيير المكوّن بالبيانات الجديدة.

لتحديث مكوّن ببيانات جديدة، يلزم أمران:

1. **الإبقاء** على البيانات بين عمليات التصيير.
2. **إطلاق** React لإعادة تصيير المكوّن بالبيانات الجديدة (إعادة تصيير).

يوفر خطاف [`useState`](/reference/react/useState) هذين الأمرين:

1. **متغير حالة** للإبقاء على البيانات بين عمليات التصيير.
2. **دالة ضبط الحالة** لتحديث المتغير وإطلاق React لإعادة تصيير المكوّن.

## إضافة متغير حالة {/*adding-a-state-variable*/}

لإضافة متغير حالة، استورد `useState` من React في أعلى الملف:

```js
import { useState } from 'react';
```

ثم استبدل هذا السطر:

```js
let index = 0;
```

بـ

```js
const [index, setIndex] = useState(0);
```

`index` متغير حالة و`setIndex` هي دالة الضبط.

> بناء الجملة `[` و`]` هنا يُسمّى [تفكيك المصفوفة](https://javascript.info/destructuring-assignment) ويسمح بقراءة قيم من مصفوفة. المصفوفة التي يعيدها `useState` دائمًا فيها عنصران بالضبط.

هكذا يعملان معًا في `handleClick`:

```js
function handleClick() {
  setIndex(index + 1);
}
```

الآن النقر على «Next» يبدّل المنحوتة الحالية:

<Sandpack>

```js
import { useState } from 'react';
import { sculptureList } from './data.js';

export default function Gallery() {
  const [index, setIndex] = useState(0);

  function handleClick() {
    setIndex(index + 1);
  }

  let sculpture = sculptureList[index];
  return (
    <>
      <button onClick={handleClick}>
        Next
      </button>
      <h2>
        <i>{sculpture.name} </i> 
        by {sculpture.artist}
      </h2>
      <h3>  
        ({index + 1} of {sculptureList.length})
      </h3>
      <img 
        src={sculpture.url} 
        alt={sculpture.alt}
      />
      <p>
        {sculpture.description}
      </p>
    </>
  );
}
```

### تعرّف على أول خطاف {/*meet-your-first-hook*/}

في React، `useState` وأي دالة أخرى تبدأ بـ"`use`" تُسمّى خطافًا (Hook).

*الخطافات* دوال خاصة متاحة فقط أثناء [تصيير](/learn/render-and-commit#step-1-trigger-a-render) React (سنتعمق في الصفحة التالية). تتيح لك «ربط» ميزات React المختلفة.

الحالة واحدة من تلك الميزات، وستتعرّف على خطافات أخرى لاحقًا.

<Pitfall>

**الخطافات—الدوال التي تبدأ بـ`use`—لا يمكن استدعاؤها إلا في المستوى الأعلى من مكوّناتك أو [خطافاتك المخصّصة.](/learn/reusing-logic-with-custom-hooks)** لا يمكن استدعاؤها داخل شروط أو حلقات أو دوال متداخلة أخرى. الخطافات دوال، لكن من المفيد اعتبارها إعلانات غير مشروطة عن احتياجات مكوّنك. «تستخدم» ميزات React في أعلى مكوّنك كما «تستورد» الوحدات في أعلى ملفك.

</Pitfall>

### تشريح `useState` {/*anatomy-of-usestate*/}

عند استدعاء [`useState`](/reference/react/useState)، تخبر React أنك تريد لهذا المكوّن أن يتذكّر شيئًا:

```js
const [index, setIndex] = useState(0);
```

في هذه الحالة، تريد أن يتذكّر React قيمة `index`.

<Note>

الاتفاق هو تسمية الزوج مثل `const [something, setSomething]`. يمكنك تسميتهما كما تشاء، لكن الاتفاقيات تسهّل الفهم بين المشاريع.

</Note>

الوسيط الوحيد لـ`useState` هو **القيمة الأولية** لمتغير الحالة. في هذا المثال، القيمة الأولية لـ`index` هي `0` عبر `useState(0)`. 

في كل مرة يصيّر فيها مكوّنك، يعطيك `useState` مصفوفة فيها قيمتان:

1. **متغير الحالة** (`index`) بالقيمة المخزّنة.
2. **دالة ضبط الحالة** (`setIndex`) التي يمكنها تحديث متغير الحالة وإطلاق React لإعادة تصيير المكوّن.

إليك كيف يحدث ذلك عمليًا:

```js
const [index, setIndex] = useState(0);
```

1. **يصيّر مكوّنك أول مرة.** لأنك مررت `0` إلى `useState` كقيمة أولية لـ`index`، يعيد `[0, setIndex]`. يتذكّر React أن `0` هي أحدث قيمة للحالة.
2. **تحدّث الحالة.** عندما ينقر المستخدم الزر، يستدعي `setIndex(index + 1)`. `index` هو `0`، فيصبح `setIndex(1)`. هذا يخبر React أن `index` أصبح `1` الآن ويطلق تصييرًا آخر.
3. **التصيير الثاني للمكوّن.** ما زال React يرى `useState(0)`، لكن لأنه *يتذكّر* أنك ضبطت `index` إلى `1`، يعيد `[1, setIndex]` بدلًا من ذلك.
4. وهكذا!

## إعطاء المكوّن عدة متغيرات حالة {/*giving-a-component-multiple-state-variables*/}

يمكنك امتلاك أي عدد من متغيرات الحالة بأي أنواع في مكوّن واحد. لهذا المكوّن متغيرا حالة: عدد `index` ومنطقي `showMore` يُبدَّل عند النقر على «Show details»:

<Sandpack>

```js
import { useState } from 'react';
import { sculptureList } from './data.js';

export default function Gallery() {
  const [index, setIndex] = useState(0);
  const [showMore, setShowMore] = useState(false);

  function handleNextClick() {
    setIndex(index + 1);
  }

  function handleMoreClick() {
    setShowMore(!showMore);
  }

  let sculpture = sculptureList[index];
  return (
    <>
      <button onClick={handleNextClick}>
        Next
      </button>
      <h2>
        <i>{sculpture.name} </i> 
        by {sculpture.artist}
      </h2>
      <h3>  
        ({index + 1} of {sculptureList.length})
      </h3>
      <button onClick={handleMoreClick}>
        {showMore ? 'Hide' : 'Show'} details
      </button>
      {showMore && <p>{sculpture.description}</p>}
      <img 
        src={sculpture.url} 
        alt={sculpture.alt}
      />
    </>
  );
}
```

#### كيف يعرف React أي حالة يعيد؟ {/*how-does-react-know-which-state-to-return*/}

ربما لاحظت أن استدعاء `useState` لا يتلقى معلومات عن *أي* متغير حالة يشير إليه. لا يُمرَّر «معرّف» إلى `useState`، فكيف يعرف أي متغير يعيد؟ هل يعتمد على سحر مثل تحليل دوالك؟ الجواب لا.

بدلًا من ذلك، لتمكين بناء الجملة المختصرة، **تعتمد الخطافات على ترتيب استدعاء ثابت في كل تصيير لنفس المكوّن.** هذا يعمل جيدًا عمليًا لأنك إن اتبعت القاعدة أعلاه («استدعِ الخطافات فقط في المستوى الأعلى»)، تُستدعى الخطافات دائمًا بنفس الترتيب. بالإضافة إلى ذلك، [إضافة linter](https://www.npmjs.com/package/eslint-plugin-react-hooks) تلتقط معظم الأخطاء.

داخليًا، يحتفظ React بمصفوفة أزواج حالة لكل مكوّن. ويحافظ على فهرس الزوج الحالي، يُضبط إلى `0` قبل التصيير. في كل مرة تستدعي `useState`، يعطيك React زوج الحالة التالي ويزيد الفهرس. يمكنك قراءة المزيد في [React Hooks: Not Magic, Just Arrays.](https://medium.com/@ryardley/react-hooks-not-magic-just-arrays-cd4f1857236e)

هذا المثال **لا يستخدم React** لكنه يعطيك فكرة عن عمل `useState` داخليًا:

```js src/index.js active
let componentHooks = [];
let currentHookIndex = 0;

// How useState works inside React (simplified).
function useState(initialState) {
  let pair = componentHooks[currentHookIndex];
  if (pair) {
    // This is not the first render,
    // so the state pair already exists.
    // Return it and prepare for next Hook call.
    currentHookIndex++;
    return pair;
  }

  // This is the first time we're rendering,
  // so create a state pair and store it.
  pair = [initialState, setState];

  function setState(nextState) {
    // When the user requests a state change,
    // put the new value into the pair.
    pair[0] = nextState;
    updateDOM();
  }

  // Store the pair for future renders
  // and prepare for the next Hook call.
  componentHooks[currentHookIndex] = pair;
  currentHookIndex++;
  return pair;
}

  };
}

  // Reset the current Hook index
  // before rendering the component.
  currentHookIndex = 0;
  let output = Gallery();

  // This is the part React does for you.

  moreButton.textContent = output.more;

  if (output.description !== null) {
    description.textContent = output.description;
    description.style.display = '';
  } else {
    description.style.display = 'none';
  }
}
```

## الحالة معزولة وخاصة {/*state-is-isolated-and-private*/}

الحالة محلية لنسخة مكوّن على الشاشة. بعبارة أخرى، **إذا صيّرت نفس المكوّن مرتين، فلكل نسخة حالة معزولة تمامًا!** تغيير إحداهما لا يؤثر على الأخرى.

هذا ما يميّز الحالة عن المتغيرات العادية التي قد تعلنها أعلى الوحدة. الحالة ليست مرتبطة باستدعاء دالة معيّن أو مكان في الشيفرة، بل هي «محلية» لموضع معيّن على الشاشة. صيّرت `<Gallery />` مرتين، فتُخزَّن حالتهما منفصلتين.

لاحظ أيضًا أن `Page` لا «تعرف» شيئًا عن حالة `Gallery` أو حتى إن كان لها حالة. بخلاف الخصائص، **الحالة خاصة تمامًا بالمكوّن الذي يعلنها.** لا يمكن للأب تغييرها. هذا يتيح لك إضافة حالة لأي مكوّن أو إزالتها دون التأثير على البقية.

ماذا لو أردت أن يبقى المعرضان متزامنين في الحالة؟ الطريقة الصحيحة في React هي *إزالة* الحالة من الأبناء وإضافتها إلى أقرب أب مشترك. الصفحات القادمة تركّز على تنظيم حالة مكوّن واحد، وسنعود لهذا في [مشاركة الحالة بين المكوّنات](/learn/sharing-state-between-components).

<Recap>

* استخدم متغير حالة عندما يحتاج المكوّن أن «يتذكّر» معلومة بين عمليات التصيير.
* تُعلَن متغيرات الحالة باستدعاء خطاف `useState`.
* الخطافات دوال خاصة تبدأ بـ`use`. تتيح لك «ربط» ميزات React مثل الحالة.
* قد تذكرك الخطافات بالاستيراد: يجب استدعاؤها بلا شروط. استدعاء الخطافات، بما فيها `useState`، صالح فقط في المستوى الأعلى لمكوّن أو خطاف آخر.
* يعيد خطاف `useState` زوجًا: الحالة الحالية والدالة لتحديثها.
* يمكنك امتلاك أكثر من متغير حالة. داخليًا، يطابق React بينها بالترتيب.
* الحالة خاصة بالمكوّن. إن صيّرته في موضعين، لكل نسخة حالتها.

</Recap>

```js
import { useState } from 'react';
import { sculptureList } from './data.js';

export default function Gallery() {
  const [index, setIndex] = useState(0);
  const [showMore, setShowMore] = useState(false);

  let hasPrev = index > 0;
  let hasNext = index < sculptureList.length - 1;

  function handlePrevClick() {
    if (hasPrev) {
      setIndex(index - 1);
    }
  }

  function handleNextClick() {
    if (hasNext) {
      setIndex(index + 1);
    }
  }

  function handleMoreClick() {
    setShowMore(!showMore);
  }

  let sculpture = sculptureList[index];
  return (
    <>
      <button
        onClick={handlePrevClick}
        disabled={!hasPrev}
      >
        Previous
      </button>
      <button
        onClick={handleNextClick}
        disabled={!hasNext}
      >
        Next
      </button>
      <h2>
        <i>{sculpture.name} </i> 
        by {sculpture.artist}
      </h2>
      <h3>  
        ({index + 1} of {sculptureList.length})
      </h3>
      <button onClick={handleMoreClick}>
        {showMore ? 'Hide' : 'Show'} details
      </button>
      {showMore && <p>{sculpture.description}</p>}
```

#### إصلاح حقول النموذج العالقة {/*fix-stuck-form-inputs*/}

عند الكتابة في الحقول، لا يظهر شيء. كأن قيم الإدخال «عالقة» بسلاسل فارغة. `value` للإدخال الأول يُضبط ليطابق دائمًا `firstName`، و`value` للثاني ليطابق `lastName`. هذا صحيح. لكليهما معالجات `onChange` تحاول تحديث المتغيرات حسب آخر إدخال (`e.target.value`). لكن المتغيرات لا تبدو أنها «تتذكّر» قيمها بين إعادة التصيير. أصلح ذلك باستخدام متغيرات حالة بدلًا من ذلك.

<Sandpack>

```js {expectedErrors: {'react-compiler': [6]}}
export default function Form() {
  let firstName = '';
  let lastName = '';

  function handleFirstNameChange(e) {
    firstName = e.target.value;
  }

  function handleLastNameChange(e) {
    lastName = e.target.value;
  }

  function handleReset() {
    firstName = '';
    lastName = '';
  }

  return (
    <form onSubmit={e => e.preventDefault()}>
      <input
        placeholder="First name"
        value={firstName}
        onChange={handleFirstNameChange}
      />
      <input
        placeholder="Last name"
        value={lastName}
        onChange={handleLastNameChange}
      />
      <h1>Hi, {firstName} {lastName}</h1>
      <button onClick={handleReset}>Reset</button>
    </form>
  );
}
```


<Solution>

أولًا، استورد `useState` من React. ثم استبدل `firstName` و`lastName` بمتغيرات حالة تُعلَن بـ`useState`. أخيرًا، استبدل كل تعيين `firstName = ...` بـ`setFirstName(...)`، ونفس الشيء لـ`lastName`. لا تنسَ تحديث `handleReset` ليعمل زر الإعادة.

```js
import { useState } from 'react';

export default function Form() {
  const [firstName, setFirstName] = useState('');

  }

  function handleLastNameChange(e) {
    setLastName(e.target.value);
  }

  function handleReset() {
    setFirstName('');
    setLastName('');
  }

  return (
    <form onSubmit={e => e.preventDefault()}>
      <input
        placeholder="First name"
        value={firstName}
        onChange={handleFirstNameChange}
      />
      <input
        placeholder="Last name"
        value={lastName}
        onChange={handleLastNameChange}
      />
      <h1>Hi, {firstName} {lastName}</h1>
      <button onClick={handleReset}>Reset</button>
    </form>
  );
}
```

#### إزالة حالة غير ضرورية {/*remove-unnecessary-state*/}

عند النقر على الزر، يفترض أن يطلب هذا المثال اسم المستخدم ثم يعرض تنبيهًا يحييه. حاولت استخدام الحالة لحفظ الاسم، لكن لسبب ما المرة الأولى تظهر «Hello, !»، ثم «Hello, [name]!» بالإدخال السابق في كل مرة بعدها.

لإصلاح الشيفرة، أزل متغير الحالة غير الضروري. (سنناقش [لماذا لم يعمل هذا](/learn/state-as-a-snapshot) لاحقًا.)
